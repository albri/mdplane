## Authentication

mdplane uses **capability URLs** for most operations - the secret is embedded in the URL path itself. No authentication headers required for `/r/`, `/a/`, `/w/` endpoints.

| URL Pattern | Permission | Description |
|-------------|------------|-------------|
| `/r/:key` | Read | GET file, structure, metadata, appends, search, stats |
| `/a/:key` | Append | Everything in Read + POST appends, heartbeats |
| `/w/:key` | Write | Everything in Append + PUT, DELETE, webhooks, keys |

Advanced endpoints (`/auth/*`, `/workspaces/*`, `/api/v1/*`) use session cookies from OAuth or API keys.
