---
title: Limits & Quotas
---

mdplane enforces limits to ensure fair usage and system stability.

---

## Content Limits

| Limit | Value | Description |
|-------|-------|-------------|
| Max file size | 10 MB | Single file size limit |
| Max append size | 1 MB | Single append size limit |
| Max appends per file | 10,000 | Total appends per file |
| Max filename length | 255 characters | Filename only |
| Max path length | 1024 characters | Full path including folders |
| Max folder depth | 10 levels | Folder nesting limit |

<Callout type="warning">
**Binary files not supported.** mdplane is text/markdown only.
</Callout>

---

## Rate Limits

### Capability URLs

| Operation | Limit | Window |
|-----------|-------|--------|
| Read | 300 requests/minute | 1 minute |
| Append | 60 requests/minute | 1 minute |
| Write | 30 requests/minute | 1 minute |
| Bootstrap | 10 requests/minute | 1 minute |

### API Keys

| Tier | Limit | Window |
|-----|-------|--------|
| Standard | 100 requests/minute | 1 minute |
| Search | 60 requests/minute | 1 minute |
| Export | 10 requests/hour | 1 hour |

### Rate Limit Response

```json
{
  "ok": false,
  "error": {
    "code": "RATE_LIMITED",
    "message": "Rate limit exceeded. Try again in 30 seconds.",
    "details": {
      "limit": 60,
      "remaining": 0,
      "resetAt": "2024-01-15T10:31:00Z"
    }
  }
}
```

## Best Practices

1. **Use pagination** — Don't fetch all results at once
2. **Cache aggressively** — Reduce redundant requests
3. **Use webhooks** — Instead of polling
4. **Export selectively** — Use paths filter
5. **Monitor usage** — Check remaining quota
